# Анализ причины: «меню снова не работает» (раздел «Проверка КД»)

## Контекст анализа
Проверил реализацию UI и API вокруг панели действий в экране **«Проверка КД»**.

Ключевой симптом из скриншота: в тулбаре есть кнопка **«🧩 Общая проверка КД»**, но действие не выполняется.

## Что обнаружено (факты по коду)

1. В компоненте `KDCheckView` кнопка «Общая проверка КД» рендерится **без обработчика** (`onClick`).
   Это означает, что пользователь кликает, но в приложении не запускается никакой сценарий.  
   
2. Для соседних кнопок обработчики есть:
   - «Верификация» -> `onRunVerification`
   - «Нейминг» -> `onRunNamingCheck`

3. В `DesignDocsWorkspace` действительно передаются обработчики только для `runVerification` и `runNamingCheck`; отдельного `runGeneralCheck`/`onRunGeneralCheck` нет.

4. В API-конфиге и backend-контроллерах отсутствует endpoint для «общей проверки»:
   - есть `/api/verification/kd`
   - есть `/api/verification/naming`
   - endpoint вида `/api/verification/general` (или аналог) отсутствует.

## Почему «снова не работает»

Это не визуальный баг и не проблема конкретных данных таблицы — это **функциональный разрыв** между UI-элементом и логикой:

- кнопка присутствует в интерфейсе,
- но цепочка `UI -> handler -> service -> API` не реализована.

Фактически кнопка сейчас — «заглушка».

## Дополнительные наблюдения

- Линтер фронтенда проходит без ошибок, т.е. это не синтаксическая поломка.
- Сборка в текущем окружении падает на создании dev-certificate в `vite.config.js`; это инфраструктурное ограничение среды и не объясняет неработающую кнопку в runtime UI.

## Вывод

Корневая причина: **кнопка «Общая проверка КД» не подключена к логике (нет `onClick`/use-case/endpoint)**. Пока не будет реализована сквозная функциональность общей проверки, поведение останется «нажал — ничего не произошло».
