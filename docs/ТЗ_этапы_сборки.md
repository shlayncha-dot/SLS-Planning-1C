# ТЗ: Экран «Этапы сборки»

## 1) Цель
Добавить в раздел **«Сборка»** новый экран/подменю **«Этапы сборки»** для распределения позиций спецификации между двумя таблицами с помощью чекбоксов и кнопок **UP/Down**.

## 2) Размещение в меню
- В главном меню **«Сборка»** добавить новый пункт: **«Этапы сборки»**.
- По клику открывается отдельный экран (страница/вкладка) «Этапы сборки».

## 3) Компоновка экрана

### 3.1 Верхняя панель действий
Сверху экрана разместить:
1. Кнопку **«Загрузить спецификацию»**.
2. Элемент выбора источника/файла таблицы (кнопка выбора файла или dropdown-выбор варианта загрузки — в зависимости от текущей архитектуры загрузки).

### 3.2 Основная область
Основную область разделить на 2 вертикальные части:
- **Левая часть — 3/4 ширины**.
- **Правая часть — 1/4 ширины**.

#### Правая часть (1/4)
- Текст: **«Введите название процедуры:»**.
- Ниже текстовое поле ввода названия процедуры.

#### Левая часть (3/4)
- Разделить по горизонтали на 2 таблицы:
  - **Верхняя таблица**.
  - **Нижняя таблица**.
- Между таблицами (в зоне управления переносом) разместить 2 кнопки:
  - **UP**
  - **Down**

## 4) Логика данных таблиц

### 4.1 Наименования таблиц
- Заголовок **верхней таблицы** должен динамически совпадать со значением поля **«Введите название процедуры»**.
- Если поле пустое — использовать заглушку, например: «Процедура (не задано)».

### 4.2 Содержимое таблиц
- **Нижняя таблица** после загрузки показывает **всю спецификацию**.
- **Верхняя таблица** изначально пустая (если не предусмотрена инициализация из сохранённых данных).

### 4.3 Первый столбец — чекбоксы
Для **обеих таблиц**:
- Первый столбец — чекбокс выбора строки.
- Чекбокс отдельный для каждой строки.
- Желательно добавить «чекбокс в заголовке» для массового выбора (опционально, если уже есть такой паттерн в проекте).

## 5) Правила переноса строк (UP/Down)

### 5.1 Кнопка UP
- Переносит отмеченные чекбоксами строки из **нижней таблицы** в **верхнюю**.
- После переноса строки удаляются из нижней таблицы и добавляются в верхнюю.

### 5.2 Кнопка Down
- Переносит отмеченные чекбоксами строки из **верхней таблицы** в **нижнюю**.
- После переноса строки удаляются из верхней таблицы и добавляются в нижнюю.

### 5.3 Общие правила
- Переносятся только строки, у которых установлен чекбокс в таблице-источнике.
- Если ничего не отмечено — действие не выполняется (без ошибки; можно показать уведомление «Не выбраны строки»).
- Дубликаты при переносе не допускаются (одна запись одновременно только в одной таблице).
- После переноса чекбоксы у перемещённых строк сбрасываются.

## 6) Требования к UI/UX
- Таблицы должны быть **компактными**:
  - минимальная высота строк,
  - минимальные внутренние отступы,
  - минимальные интервалы между блоками,
  - приоритет отображения максимального количества строк на экране.
- Сохранить читаемость и соответствие существующему дизайн-стилю приложения.

## 7) Валидации и состояния
- Кнопки UP/Down активны только при наличии выделенных строк в соответствующей таблице-источнике (или допускается активное состояние с мягкой обработкой пустого выбора).
- При загрузке спецификации:
  - очистить текущие данные в обеих таблицах,
  - заново заполнить нижнюю таблицу данными из загруженной спецификации.

## 8) Критерии приёмки
1. В меню «Сборка» доступен новый пункт «Этапы сборки».
2. На экране есть верхняя панель с кнопкой «Загрузить спецификацию» и выбором загрузки таблицы.
3. Экран делится на левую (3/4) и правую (1/4) части.
4. Справа отображаются текст «Введите название процедуры:» и поле ввода.
5. Слева отображаются две таблицы (верх/низ) и между ними кнопки UP/Down.
6. Заголовок верхней таблицы равен введённому названию процедуры.
7. В нижней таблице отображается вся загруженная спецификация.
8. В обеих таблицах первый столбец — чекбоксы по строкам.
9. Кнопка UP переносит отмеченные строки снизу вверх.
10. Кнопка Down переносит отмеченные строки сверху вниз.
11. Табличный интерфейс выполнен в компактном виде (минимальные интервалы/высоты).

## 9) Открытые уточнения (если потребуется перед реализацией)
1. Формат «выбора загрузки таблицы»: файл с диска, выбор из БД или оба режима?
2. Нужна ли кнопка «Сохранить этапы» и где хранить результат распределения?
3. Требуется ли сортировка/фильтрация в таблицах?
4. Нужен ли мультивыбор «Выбрать все» в заголовке?
5. Нужно ли сохранять введённое название процедуры между сессиями?
